# Contributing to Fiszki

Thank you for your interest in contributing to Fiszki! Whether you're a developer, designer, translator, or just someone with great ideas, we welcome your contributions.

## Reporting Bugs and Suggesting Features

If you encounter a bug or have an idea for a new feature, please open an issue on our [GitHub Issues](https://github.com/magicmychal/Fiszki/issues) page. When reporting a bug, include as much detail as possible:

- A clear and descriptive title.
- Steps to reproduce the issue.
- Expected behavior.
- Actual behavior.
- Screenshots (if applicable).
- Your device and Android version.

For feature requests, describe the feature you'd like to see and explain why it would be useful.

## Contributing Translations

We welcome translations to make Fiszki available in more languages! Here's how you can help:

1. **Fork the repository** and clone it to your local machine.
2. Navigate to the `app/src/main/res` directory.
3. Create a new directory for your language (e.g., `values-es` for Spanish).
4. Copy the `strings.xml` file from the `values` directory into your new directory.
5. Translate the strings in the new `strings.xml` file.
6. Commit your changes and push them to your fork.
7. Open a pull request with your changes.

If you're not comfortable with GitHub, you can also send us the translated `strings.xml` file via email or another method, and we'll handle the rest.

## Setting Up the Development Environment

To contribute code, you'll need to set up the development environment:

1. **Fork the repository** and clone it to your local machine.
2. Open the project in Android Studio.
3. Ensure you have the latest version of Android Studio and the necessary SDKs installed.
4. Build the project to resolve dependencies.

## Submitting Pull Requests

Once you've made your changes, follow these steps to submit a pull request:

1. Commit your changes with a clear and descriptive commit message.
2. Push your changes to your fork on GitHub.
3. Open a pull request from your fork to the main repository.
4. Provide a description of your changes and any relevant context.

We'll review your pull request as soon as possible and provide feedback if needed.
